# Per-Class Study Hub Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans or subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the flat study hub into a scalable, auto-maintained archive — a hub of class cards, each linking to a per-class page that lists that class's exam/topic reviews — with `assemble_quiz.py` registering each quiz into both levels idempotently.

**Architecture:** Static HTML reusing the existing dark theme. Two bundled templates (`hub`, `class-page`). `assemble_quiz.py` gains a `--hub` flag and, from `class/exam/kind` metadata, (1) creates/updates the class page (exam sections + quiz cards) and (2) creates/updates the class card on the hub — all via the existing marker-based string-surgery, idempotent by `href` (cards) and `classSlug` (class cards).

**Tech Stack:** Python 3 (stdlib only), static HTML/CSS. Spec: [docs/superpowers/specs/2026-06-22-per-class-study-hub-design.md](../specs/2026-06-22-per-class-study-hub-design.md).

## Global Constraints

- Skill dir: `.claude/skills/nclex-quiz-builder/`; workspace (gitignored): `…-workspace/`.
- Idempotent registration: re-running a build updates entries, never duplicates.
- Static only — no JS framework, no build server (GitHub Pages).
- UTF-8 output; ASCII hyphens in console messages.
- Reuse existing CSS design tokens / `.card` / `.chip` / `.btn` classes; match the existing dark theme.
- Existing `maternal-child-weeks4-7.html` URL must not change.
- `cat`/quiz-item behavior and the existing linter are unchanged by this work.

def slugify(text):
    s = re.sub(r"[^a-z0-9]+", "-", (text or "").lower()).strip("-")
    return re.sub(r"-{2,}", "-", s)
